<objective>
Collapse the over-structured `src/services/api-gap/` directory (3 files for what is effectively a frozen 6-element constant) into a single `src/services/api-gap.ts` file. Promote the `API_GAP_ENTRIES` constant + `ApiGapEntry` type to `src/domain/api-gap/catalog.ts` (pure data belongs in domain per the lite-hexagonal layering rule). Rewrite all 7 importers. Add a CI grep gate that forbids future regressions.

Purpose: today's structure is `src/services/api-gap/{index,data,types,index.test}.ts` — 3 source files plus a test for a 6-entry frozen readonly array. The directory is the wrong tier for the data (pure constant, no I/O — belongs in domain) and the wrong shape for the accessor (one async function — belongs in a single flat file). ARCH-08 cleans up both.

Output: 2 created files (`domain/api-gap/catalog.ts` + `services/api-gap.ts`), 1 renamed test (`services/api-gap/index.test.ts` → `services/api-gap.test.ts`), 4 deleted files (the entire `services/api-gap/` directory), 7 importers rewritten, 1 new grep gate. PR `refactor/10-arch-08-api-gap-inline` lands on its own branch off the latest `main`, merged via GitHub PR with explicit user approval.

Scope: pure structural refactor. Zero logic changes. The `getApiGap()` async accessor returns `{ entries: API_GAP_ENTRIES }` byte-for-byte identical to today. The 6 frozen entries move with their `Object.freeze` wrapper intact. The load-bearing parity test (`tests/contract/api-gap-md-parity.test.ts`) — which asserts the markdown table generated by `scripts/generate-api-gap-md.ts` matches the constant — continues to pass.

This plan depends on plan 10-02 (NOT 10-03 or 10-04) because the api-gap work is independent of doctor wiring. After plan 10-02 ships, plans 10-03/10-04/10-05 can technically run in any order, but the build-order convention per RESEARCH §"Recommended PR boundaries" places api-gap last (cosmetic; saves it for the "we shrunk something" closing note).
