<objective>
TDD plan: ship the four pure-domain functions that consume Plan 04-03's statistics primitives:
- `computeBaseline` — per-metric trailing-30 median + raw MAD + n + coverage_pct (D-02 anchor — service supplies the already-windowed values; domain stays pure).
- `computeZAnalysis` + `selectAnomalies` — Z-score gating + D-06 firing rule + direction lookup.
- `ANOMALY_DIRECTION` — per-metric direction map (D-06).
- `confidenceFromCounts` — D-13 tier-gating function (emits the D-10 `'insufficient'` tier when scoredDays < 10).

Purpose: this is the chassis daily review composes. REV-01 (trailing-30 median + MAD on SCORED-only with DST exclusion — D-02 trailing-30 anchor from reviewed_date) lands here. REV-02 (confidence-tier gate, insufficient/weak/strong + Z refused < 14 days) lands here. REV-05 (insufficient → states what is missing and declines) is unlocked once `confidenceFromCounts` + the `ZAnalysis.refused` discriminated-union path exist; the D-10 typed positive output (empty anomalies/actions + populated insufficient_reason) is composed by Plan 04-07 over this layer's primitives.

Pitfall 5 ("mixed-recency Z-refusal — per-metric vs per-window") is the key constraint: the daysAvailable count is PER METRIC (HRV may have 12 days while sleep has 22), not per window. Wave 2's daily.ts service is where the per-metric assembly happens; this plan exposes the primitive that consumes the per-metric count.

Output: 4 source files + 4 test files. ~150 LOC of pure domain.
</objective>

  **confidence/index.ts** (RESEARCH §3 verbatim; D-10 anchors the 'insufficient' tier emission):
  ```
  export function confidenceFromCounts(opts: {
    scoredDays: number;
    windowDays: number;
  }): ConfidenceGate {
    const coveragePct = (opts.scoredDays / opts.windowDays) * 100;
    if (opts.scoredDays < 10) return { tier: 'insufficient', coveragePct, minRequired: 10, sampleSize: opts.scoredDays };
    if (opts.scoredDays >= 20 && coveragePct >= 70) return { tier: 'strong', coveragePct, minRequired: 20, sampleSize: opts.scoredDays };
    return { tier: 'weak', coveragePct, minRequired: 10, sampleSize: opts.scoredDays };
  }
  ```

  **confidence/index.test.ts:**
  - `scoredDays = 0` → insufficient.
  - `scoredDays = 9, windowDays = 30` → insufficient (under threshold).
  - `scoredDays = 10, windowDays = 30` → weak (boundary).
  - `scoredDays = 19, windowDays = 30` → weak.
  - `scoredDays = 20, windowDays = 30, coverage 66.7%` → weak (≥ 20 but coverage < 70%).
  - `scoredDays = 21, windowDays = 30, coverage 70%` → strong (≥ 20 AND coverage ≥ 70).

  **baselines/index.ts:**
  - Exports `computeBaseline(values: number[], windowDays: number): BaselineStats | null` where `metric` is provided by the caller (the function returns a single-metric result; the caller iterates METRIC_NAMES). Actually: simpler shape — `computeBaseline(metric: MetricName, values: number[], windowDays: number): BaselineStats`. Returns:
    - `{ metric, median: median(values), mad: medianAbsoluteDeviation(values), n: values.length, coverage_pct: (values.length / windowDays) * 100 }`
  - When `values.length === 0`: throws (caller is responsible for refusing upstream — same discipline as median/mad).
  - The function consumes ALREADY-FILTERED values (non-null, finite, SCORED-only, non-DST-excluded) AND ALREADY-WINDOWED to the trailing-30 from reviewed_date (D-02 anchor — windowing happens at the SERVICE layer in Plan 04-07's `getDailyReview` via `resolveReviewedDate` + `cycles.byRange(reviewed_date - 29d, reviewed_date)`). Domain code never sees raw rows.

  **baselines/index.test.ts:**
  - Empty values array throws.
  - `computeBaseline('recovery_score', [50, 60, 70, 80, 90], 30)` returns `{ metric: 'recovery_score', median: 70, mad: 10, n: 5, coverage_pct: 16.67 }` (deviations [20, 10, 0, 10, 20], MAD = 10).
  - 30 identical values [50 × 30] → `{ median: 50, mad: 0, n: 30, coverage_pct: 100 }` (MAD = 0 edge).
  - Different metric names — function preserves the `metric` slot verbatim from the input.
  - coverage_pct precision: 9 scored days in 30 → coverage_pct ≈ 30; the formatter applies any rounding (this layer returns raw float).

  **anomalies/anomaly.ts:**
  - Exports `computeZAnalysis(input: { value: number | null; baseline: BaselineStats; daysAvailable: number; daysRequired: 14 }): ZAnalysis`:
    - If `input.value === null` OR `!Number.isFinite(input.value)`: returns `{ kind: 'refused', reason: 'insufficient_days', days_available: input.daysAvailable, days_required: 14 }` (defensive — though caller should pre-filter).
    - If `input.daysAvailable < 14`: returns `{ kind: 'refused', reason: 'insufficient_days', days_available: input.daysAvailable, days_required: 14 }`.
    - If `input.baseline.mad === 0`: returns `{ kind: 'refused', reason: 'baseline_mad_zero', days_available: input.daysAvailable }`.
    - Otherwise: `robust_sigma = 1.4826 × input.baseline.mad; z = (input.value - input.baseline.median) / robust_sigma;` returns `{ kind: 'computed', value: z, baseline_median: input.baseline.median, baseline_mad: input.baseline.mad, tier: input.daysAvailable >= 20 ? 'strong' : 'weak' }`.
  - Exports `selectAnomalies(input: { today: TodayMetrics; baselines: BaselineStats[]; perMetricDaysAvailable: Record<MetricName, number> }): Anomaly[]`:
    - For each metric in METRIC_NAMES:
      - Look up baseline (skip metric if not present).
      - Read `today[metric]` (may be null).
      - Call `computeZAnalysis({ value: today[metric], baseline, daysAvailable: perMetricDaysAvailable[metric], daysRequired: 14 })`.
      - If result.kind === 'refused': skip (no Anomaly).
      - Otherwise: read `direction = ANOMALY_DIRECTION[metric]`; if bidirectional → skip (informational only, NOT actionable per D-06).
      - Fire Anomaly when `|result.value| >= 2.0` AND (`(direction === 'low' && result.value <= -2.0) || (direction === 'high' && result.value >= +2.0)`).
      - Emit `{ metric, z: result.value, direction, baseline_median: result.baseline_median, baseline_mad_scaled: 1.4826 × result.baseline_mad, tier: result.tier }`.
    - Returns the Anomaly[] (may be empty per ADR-0004 — empty is a typed positive output).

  **anomalies/anomaly.test.ts:**

  - Z-refused: baseline.mad = 0 → refused.baseline_mad_zero.

  - Anomaly NOT firing — day_strain (bidirectional): even with |z| = 3, NO Anomaly emitted (bidirectional = informational only).

  - Empty case — today all-favorable z's → returns `[]` (ADR-0004 typed positive output).
