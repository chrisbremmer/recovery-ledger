<objective>
Ship the build-time API-gap markdown generator (D-17 + D-18 + D-19). The generator turns the Phase 4 D-28 source-of-truth `API_GAP_ENTRIES` array into a human-readable markdown file under `docs/install/api-gap.md`. A parity contract test asserts the markdown stays in lockstep with the TS source — a developer who edits `data.ts` without regenerating loses CI.

Purpose: DOC-03 requires the API-gap doc to be available to humans (the install guide links it). Phase 4 already gave us the canonical data; Phase 5 ships the second view onto the same data, with a forcing function preventing drift.

Output: pure render function + thin CLI wrapper + committed markdown + parity contract test + npm script. No new MCP surface (the markdown is for humans; the MCP tool already returns the same data structurally).
</objective>

        export function renderApiGapMarkdown(entries: readonly ApiGapEntry[]): string {
          const header = [
            '<!-- Generated from src/services/api-gap/data.ts — do not hand-edit. -->',
            '<!-- Run `npm run docs:generate-api-gap` after changing the source.   -->',
            '',
            '# WHOOP API v2 Gap',
            '',
            'WHOOP consumer-app features that are NOT exposed via the public v2 API.',
            '',
          ];
          const body = entries.flatMap((entry) => [
            `## ${entry.feature}`,
            '',
            `**WHOOP app path:** ${entry.whoop_consumer_path}`,
            '',
            `**Available via v2 API:** No.`,
            '',
            `**Closest v2 alternative:** ${entry.alternative_via_v2 ?? 'None.'}`,
            '',
            entry.notes,
            '',
          ]);
          // Join with \n and ensure exactly one trailing newline. Per RESEARCH §Finding 5,
          // idempotency requires deterministic newline handling — no double-trailing.
          return [...header, ...body].join('\n').replace(/\n+$/, '') + '\n';
        }

          const content = renderApiGapMarkdown(API_GAP_ENTRIES);
